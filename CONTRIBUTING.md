# Contributing to TU-8 Framework

## Overview

This is a principle-level reference framework. 
Contributions should focus on improving clarity and accessibility, not changing the underlying principles.

## What to Contribute

### ✅ Welcome
- Translation improvements
- Documentation clarity fixes
- Spelling and grammar corrections
- Additional real-world examples that illustrate existing principles

### ❌ Not Acceptable
- Changing the core principles (this is a reference framework, not a sandbox)
- Adding new principles without clear rationale
- Removing existing principles
- Commercial use of the framework

## Contribution Process

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create a branch** for your changes (`git checkout -b improve/clarity-fix`)
4. **Make changes** in `/docs` folder
5. **Test** — ensure all documents render correctly in Markdown
6. **Commit** with clear message (`git commit -m 'fix: improve explanation of core layer isolation'`)
7. **Push** (`git push origin improve/clarity-fix`)
8. **Open Pull Request** with description of changes

## Document Standards

- Use Simplified Chinese for framework documents
- Use English for navigation and README
- Keep paragraphs under 5 lines
- Use tables for structured information
- Include example boxes (> blockquote) for complex concepts
